# 2026-05-13 开发日志

## Issue 描述置顶横幅 (vue)

### 背景

新 Issue 实体取代了原 Todo 看板,issue 有 `description` 字段,但前端选中 issue 过滤消息流后,只能看到归属该 issue 的消息,看不到 issue 自身的描述/上下文。`IssueCard` 也没有描述编辑入口。

### 方案

参考 Telegram 频道置顶,采用置顶横幅而非伪 message 条目:

- 不放进消息流,避免与日期分隔符、合并选择、滚动恢复等逻辑打架(issue 描述没有时间轴语义位置)。
- 取消筛选时整条横幅消失,无需从流里抽离脏数据。
- 位置紧贴左侧 feed header 下方,与"当前筛选上下文"语义一致。

### 实现

**新增组件** `vue/src/components/IssuePinnedBanner.vue`:

- 折叠态:标题 + `message_count` 徽标 + 描述(markdown 渲染,默认 `line-clamp-2`),挂载后用 `scrollHeight > clientHeight` 判定是否显示「展开」按钮。
- 编辑态:标题输入框 + `VditorEditor`(复用 MessageComposeInline 用法),保存调用 `PATCH /issues/{id}`,成功后 emit `updated`,父组件原地替换 `issues[]` 中的条目,无需重新拉取。
- 操作区:编辑、取消筛选(emit `clear`,父组件调用 `selectIssue(null)`)。

**接入** `vue/src/views/Message.vue`:

- 新增 `selectedIssue` computed 从 `issues.value` 按 `selectedIssueId` 查找当前 issue 对象。
- 横幅插入在「消息流」搜索 header 与滚动区之间,`v-if="selectedIssue"`。
- `onIssueUpdated(updated)` 原地替换 `issues[]` 对应项,保持 `selectedIssue` 计算属性即时刷新。

### 顺带补齐

- IssueCard 之前没有描述编辑入口,本次横幅的编辑按钮成为补充入口。

## briefing × issue 联动

### 背景

briefing skill 每天生成晨间简报后，只是把整篇汇报写进笔记库，与 Issue 系统脱钩。希望昨日动态能"推进"到具体的进行中 issue 时间线上，让 issue 卡片自然累积上下文与实现方案建议。

### 改动

**MCP (`backend/mcp_server.py`)**

- `write_message` 新增可选参数 `issue_id`，写笔记时直接归到对应 issue 时间线（backend `POST /messages` 已支持该字段）。
- 新增 `list_issues(status="doing")`：返回 issue 列表（id/title/description/message_count），供 briefing 拿到候选清单。
- 新增 `read_issue(issue_id, history_limit=5)`：聚合 `GET /issues/{id}` + `GET /messages/with-detail?issue_id=...&limit=N`，一次返回 issue 详情和最近若干条关联消息，避免重复推送昨日已记录的结论。
- 同步更新 server `instructions`，让 agent 感知新工具。

**SKILL (`.claude/skills/briefing/SKILL.md`)**

- 简报落库后追加「Issue 进展联动」流程：`list_issues` → 用昨日邮件/Teams/文件回顾匹配相关性 → `read_issue` 查重 → `write_message(issue_id=..., text=结构化更新)`。
- 强调只对有明显信号的 issue 推送，不硬凑内容。
- 简报模板「待处理事项」后追加「Issue 进展更新」小节，列出本次更新过的 issue。
