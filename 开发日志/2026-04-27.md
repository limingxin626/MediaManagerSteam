## 视频预览（章节）功能

### 背景

视频详情页的「预览片段」中栏之前是占位文案「暂无片段」。本次落地用户标记的多预览图（章节）：一段视频可关联 0..N 个预览图，每张带 `frame_ms` 必填 + 可选 `start_ms` / `end_ms`，前端按时间顺序展示，点击跳转到对应区间播放。

来源两条：
1. 播放页面截图 → canvas 出图 → 上传成新 image media → 后端写入 `frame_ms` 等字段并归属当前视频。
2. 从已有图片中挑（picker） → 用户填时间 → 后端只更新归属字段。

### 数据模型决策

**单表 1:N 严格独占方案**：直接在 `media` 表加 4 个可空字段（`video_media_id` 自引用 FK + `frame_ms` / `start_ms` / `end_ms`）。一张图片 Media 行只能属于一个视频的一个区间；想多处复用就重新截图/上传（Blake2b 仅保证文件不重复，行可以重复——所以本次同时去掉了 `file_hash` 的 unique 约束）。

预览 Media 行**默认在 `/media` 列表与 `/media/timeline` 中隐藏**（添加 `video_media_id IS NULL` 过滤），避免和普通图片混淆。`/media/feed` 走 MessageMedia join，预览图不出现在 MessageMedia 中，自然过滤。

CASCADE：删视频 → 自动清掉所有挂载的预览 Media 行；删预览图独立删行（路由层）。SQLite FK 通过 connect 事件 `PRAGMA foreign_keys=ON` 启用。

### 后端实现

- **迁移 `backend/alembic/versions/20260427_1200_a1b2c3d4e5f6_add_video_preview_fields.py`**：批量加 4 列 + 自引 FK CASCADE + 组合索引 `ix_media_video_frame`，并把 `ix_media_file_hash` 从 unique 改为普通索引。
- **模型 `backend/app/models/__init__.py`**：`Media` 加 4 字段、self relationship `video` + backref `previews`（`order_by=frame_ms`、`cascade=all, delete-orphan`、`passive_deletes=True`）。新增 engine `connect` 事件启用 SQLite 外键。
- **Schemas `backend/app/schemas/media.py`**：`MediaResponse` 加 4 字段；新增 `VideoPreviewItem`（继承 `MediaUrlMixin` 自动得到 `file_url` / `thumb_url`）/ `VideoPreviewCreate` / `VideoPreviewUpdate`。
- **Service `backend/app/services/media_service.py`**：把 `process_file` 中"hash + ffprobe + 缩略图 + 入库"部分抽成 `process_standalone_file(db, path, skip_dedup=False)`；新增 `create_preview_media(db, path)` 走 `skip_dedup=True` 强制建新行（独占语义）。`process_file` 改为先调 standalone 再 `_link_media`，对外行为不变。
- **Router `backend/app/routers/media.py`**：5 个新端点
  - `GET /media/{id}/previews` 按 `frame_ms` 升序列表
  - `POST /media/{id}/previews` 把已有 image 设为预览（独占校验 → 409）
  - `POST /media/{id}/previews/screenshot`（multipart：`file` + `frame_ms` + 可选 `start_ms`/`end_ms`）截图上传创建新行
  - `PATCH /media/previews/{preview_id}` 部分更新时间字段
  - `DELETE /media/previews/{preview_id}` 删除一条预览（顺带删缩略图与文件）
  - 扁平路径 `/previews/{preview_id}` 必须**声明在 `/{media_id}` 之前**否则会被 int 路径吞掉
- 列表过滤：`/media`（else 分支）、`/media/timeline` 加 `Media.video_media_id.is_(None)`。

## 视频详情页截图流程优化

### 背景

之前 `VideoPreviewStrip.vue` 的「截图设为预览」是「点一下立刻 canvas → 上传」一气呵成，没有给用户预览/调整的机会。需要：截图后先在前端展示，让用户确认 + 填写起止时间后再上传；而且截图自动归到该视频对应的消息里。

### 实现内容

**后端 — `backend/app/routers/media.py`（`add_preview_from_screenshot`）**

- 在新建预览 Media 行写完 `video_media_id` / `frame_ms` / `start_ms` / `end_ms` 后，`db.flush()` 拿到截图 media 的 id
- 查 `MessageMedia` 找父视频被引用的所有消息，按 `created_at asc, id asc` 取**最早一条**（与用户确认的策略：多条引用时仅插入最早消息）
- `position` = 该消息当前 `MAX(position) + 1`，再 `db.add(MessageMedia(...))`
- 视频未被任何消息引用时跳过，行为同此前

**前端 — `vue/src/components/VideoPreviewStrip.vue`**

- 新增 `pendingShot: { blob; dataUrl } | null` 与 `shotForm`（frame/start/end_ms）状态
- `captureScreenshot`：仅做 canvas → `toBlob` + `toDataURL`，把结果存进 `pendingShot`、把 `shotForm.frame_ms` 预填为当前播放位置毫秒，**不再上传**
- 新增 `confirmShot`：组装 FormData（仅在非 null 时附带 `start_ms`/`end_ms`），fetch 上传，成功后清 `pendingShot`、刷新预览列表
- 新增 `cancelShot`：业务进行中禁用，否则丢弃 blob 关闭 modal
- 新增确认 modal：顶部 `<img :src="pendingShot.dataUrl">` 展示 JPEG 截图预览，下方三个数字输入框（frame_ms 时间戳、start_ms 开始、end_ms 结束）
- 「截图设为预览」按钮的 `:disabled` 加上 `pendingShot != null`，避免在确认 modal 已开时再次截图覆盖

### 涉及文件

- `backend/app/routers/media.py`
- `vue/src/components/VideoPreviewStrip.vue`

### 验证

- `pnpm exec vue-tsc --noEmit` 通过
- 端到端点击验证未跑（同上一节，`DATA_ROOT` 未在该 shell 设置）。下次启动后请按以下手测：
  1. 进任意视频详情页点截图，确认 modal 显示当前帧画面

## 截图确认 UI 改为内联 + 起止时间按钮 + 上传后滚动续接

### 背景

上一版的「确认截图」是 fixed 全屏 Teleport modal + 三个手动数字输入框，不便操作。需要：
1. 不要弹窗，改为在中栏（信息栏「预览片段」列表）下方原地显示
2. 不再让用户手填毫秒数，改为提供「设为开始」/「设为结束」两个按钮，点击即把当前播放位置写入对应字段
3. 上传成功后，把刚刚的 `end_ms` 滚动到下一次截图的 `start_ms`，`end_ms` 置空——配合连续截多张组成区间链。

### 实现内容

**前端 — `vue/src/components/VideoPreviewStrip.vue`**

- 删除 Teleport 全屏截图 modal，替换为内联卡片，渲染位置：action buttons → preview list → **pending shot card** → 其它 modal。卡片宽度跟随中栏宽度，不影响 picker / edit 这两个仍是 modal 的流程。
- 卡片内容：截图 dataUrl 大图、三行只读时间显示（时间戳 / 开始 / 结束，未设置时灰字「未设置」+ 文本对齐用 `font-mono`）、两组按钮行：
  - 「设为开始」「设为结束」：`shotForm.start_ms = currentMs()` / `shotForm.end_ms = currentMs()`，依赖 `videoEl` 可用
  - 「取消」「确认上传」
- `confirmShot` 加三段大小关系校验（任一不满足即 toast 报错并 return，不发起请求）：
  - `start_ms != null && start_ms > frame_ms` → 「开始时间必须 ≤ 时间戳」
  - `end_ms != null && end_ms < frame_ms` → 「结束时间必须 ≥ 时间戳」
  - `start_ms != null && end_ms != null && start_ms > end_ms` → 「开始时间必须 ≤ 结束时间」
- 上传成功后：`pendingShot = null`、`shotForm.start_ms = 上一次的 end_ms`、`shotForm.end_ms = null`。下次按「截图设为预览」时 `start_ms` 已经预填好，连续截多张时不必重复设。
- `cancelShot` 同时清掉 `start_ms`/`end_ms`，避免取消后残留旧值。
- `captureScreenshot` 不再每次重置 `start_ms`/`end_ms`，只刷新 `frame_ms`——给上传后续接的滚动逻辑留路。

### 涉及文件

- `vue/src/components/VideoPreviewStrip.vue`

### 验证

- `pnpm exec vue-tsc --noEmit` 通过
- 端到端未手测，下次启动后请按以下手测：
  1. 截图按钮点一下，卡片在「预览片段」列表下方原地出现，时间戳是当前播放位置
  2. 拖动播放位置 → 点「设为开始」/「设为结束」，对应行立刻显示更新后的时间
