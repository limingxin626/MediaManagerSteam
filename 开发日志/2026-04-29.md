## 消息编辑允许 text 为空

### 背景

编辑消息时，若用户清空文本只保留媒体/标签，前端将 `text` 转换为 `null` 提交，后端 `MessageUpdate` 中 `if update_data.text is not None` 跳过赋值，导致空文本无法被持久化。

### 实现内容

**前端 — `vue/src/views/Message.vue`**

- `onDialogUpdated` 中提交 PATCH 时，`text: text || null` → `text`，原样发送空字符串，后端会写入 `message.text = ""`。
- 创建模式（`MessageComposeDialog.vue` 第 269 行）保持 `|| null`，因为 `canSubmit` 仍要求 text 或 files 至少一项。

### 涉及文件

- `vue/src/views/Message.vue`

### 验证

- 编辑现有消息，清空文本框后保存：列表与详情面板的 text 显示为空，标签与媒体保留。

## 修复合并带媒体消息时的外键报错

### 背景

`POST /messages/merge` 合并两条均含 media 的消息时返回 500：`sqlite3.IntegrityError: FOREIGN KEY constraint failed`。

### 原因

`Message.message_media` 配置了 `cascade="all, delete-orphan"`。原实现直接给 `MessageMedia.message_id` 赋值为 target 的 id，但这些行仍挂在 `other.message_media` 集合中。随后用 `db.query(Message).filter(...).delete(synchronize_session="fetch")` 触发 ORM 同步，将关联的 message_media 视为 orphan 一并删除/更新，与未刷新的状态冲突，违反 FK。

### 实现内容

**`backend/app/routers/message.py` `merge_messages`**

- 通过 `other.message_media.remove(rel)` + `target.message_media.append(rel)` 经由关系集合迁移，避免 orphan 误判。
- 显式清空 `other.tags = []` 解除 message_tag 关联。
- `db.flush()` 后改用 `db.delete(other)`（逐条 ORM 删除，触发正确的 cascade 顺序），不再走 bulk delete。

### 涉及文件

- `backend/app/routers/message.py`

### 验证

- 重启后端，前端选择两条均含媒体的消息执行合并：返回 200，目标消息合并所有媒体并按顺序排列，其余消息删除。

## 媒体文件替换功能

### 背景

在 MediaDetail 预览页希望支持用一个本地文件替换当前 media 对应的实际文件，同时保留 Media 行的 id（以及所有 MessageMedia 链接、标签、视频预览等关联）。

### 实现内容

**后端 — `backend/app/services/media_service.py`**

- 新增 `replace_media_file(db, media_id, src_path)`：先计算新文件 hash，若与另一条 Media 重复则走合并分支：通过 `_merge_media_into` 把当前 Media 的 `MessageMedia`（同一 message 内若已存在 dst 则丢弃 src 链接，避免 `uq_mm_message_position` 冲突）、`media_tag`、以及 `Media.video_media_id` 指向自身的预览帧统一迁移到 dst（dst 非视频时连同预览文件/缩略图一并删除），最后删除当前 Media 行及其文件/缩略图，返回 dst。无重复则原地覆盖：同扩展名直接覆盖原 `file_path`，扩展名变化时改写为新扩展名并删除旧文件，重新计算 `file_hash`/`file_size`/`mime_type`/`width`/`height`/`duration_ms`，重生成缩略图。允许跨类型替换（视频↔图片），但拒绝替换视频预览帧（`video_media_id is not None`）。

**后端 — `backend/app/routers/media.py`**

- 新增 `POST /media/{media_id}/replace`（multipart `file`）：将上传文件落到 `UPLOAD_DIR/YYYY/MM/DD/replace_{id}_{ts}{ext}` 临时位置，调用 `replace_media_file`，无论成功失败都清理临时文件，返回更新后的 `MediaResponse`。

**前端 — `vue/src/composables/useApi.ts`**

- `request` 增加 `body instanceof FormData` 分支，避免对 FormData 套 `JSON.stringify` 并保留浏览器自动添加的 `Content-Type: multipart/form-data; boundary=...`。

**前端 — `vue/src/components/MediaPreview.vue`**

- 顶部操作栏（旋转按钮左侧）加「替换」按钮 + 隐藏 `<input type="file">`，点击触发文件选择，用 `FormData` 调用 `/media/{id}/replace`。
- 替换成功后将更新后的字段直接写回 `props.items[currentIndex]`（包括可能变化的 `id`，因为去重合并会返回另一条 Media），并维护 `mediaCacheBust` 映射；`getMediaUrl` 在原始 URL 上拼接 `?t=ts` 强制刷新视频/图片缓存；`thumb_url` 也带上时间戳。合并发生时 toast 提示「已合并到已有媒体」。
- 新增 `media-replaced` 事件供父组件可选监听（用于刷新缩略图栏外的列表缓存）。

- `vue/src/composables/useApi.ts`
- `vue/src/components/MediaPreview.vue`
