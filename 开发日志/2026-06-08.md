# 2026-06-08 Mac 消息页 markdown 渲染(分两步)

## 上午:仅 inline markdown(已被覆盖)

最早实现是 SwiftUI 原生 `AttributedString(markdown:)`,选项 `inlineOnlyPreservingWhitespace`。
仅支持 `**粗体**` / `*斜体*` / `` `行内代码` `` / `[文本](url)` / `~~删除线~~`,
块级语法(标题/列表/引用/代码块)按字面保留。Commit `9fad817`。

用户反馈"连标题都没有,太少",决定升级到全量 markdown。

## 下午:接入 swift-markdown-ui(本期最终方案)

### 调研结论

| 方案 | 维护 | 工作量 | 备注 |
|---|---|---|---|
| MarkdownUI (gonzalezreal) | ⚠️ maintenance mode | 加 SPM + 几行 | GFM 全特性,作者承诺修 bug;新开发迁去 Textual |
| apple/swift-markdown 自写 renderer | ✅ 活跃 v0.8.0 | 200-400 行块级 View | 全自维护 |
| 自己手工拆块 | / | 100+ 行 | 不支持代码块/表格 |

选 MarkdownUI:**最小代码量拿到 GFM 全集**。即便将来要换,SwiftUI 视图层换成自写 swift-markdown 渲染也容易。

### 改动

#### 新文件

- `MyNote/MyNote/MarkdownBody.swift` — `struct MarkdownBody: View`,包住 `MarkdownUI.Markdown(_:)`:
  - 基础正文 14pt 对齐原 inline 渲染视觉
  - 行内代码:等宽 + 浅灰底
  - 链接:accent color(沿用原 NSDataDetector 着色)
  - H1-H3 自定义字号(消息卡片密度,不要太大)
  - 围栏代码块:浅底 + 圆角 + 内边距
  - 引用块:左侧粗竖线 + 灰字
  - `.textSelection(.enabled)` 让正文可复制

#### 删除

- `MyNote/MyNote/MessageTextRenderer.swift` — 上午加的 inline-only renderer,完整被 MarkdownBody 取代

#### 改造

- `MyNote/MyNote/MessageCard.swift` — `textBody(_:)` 改调 `MarkdownBody(text:)`
- `MyNote/MyNote/MessageDetailPane.swift` — 同款改动

#### 配置

- `MyNote/MAC_TODO.md` — 新增 1.1.5 节:**Xcode 里手动添加 SPM 包 https://github.com/gonzalezreal/swift-markdown-ui(版本 2.4.1+)**,不加 app 编不过(`import MarkdownUI` 失败)

### 不动

- Xcode project 文件本身:`PBXFileSystemSynchronizedRootGroup`,新 `.swift` 自动入 target;SPM 包仍然要 Xcode GUI 操作
- vue / Android / backend 零改动

## 现在支持的范围

| 特性 | 支持 | 备注 |
|---|---|---|
| H1-H6 标题 | ✅ | H1-H3 自定义字号,H4-H6 走 default theme |
| 有序 / 无序 / 任务列表 | ✅ | GFM 任务列表 `- [ ]` 也支持 |
| 引用块 | ✅ | 左侧粗竖线样式 |
| 围栏代码块 | ✅ | 浅底 + 圆角,等宽字体 |
| 行内代码 | ✅ | 浅灰底 |
| 加粗 / 斜体 / 删除线 | ✅ | |
| 链接 | ✅ | accent color 高亮(不再需要 NSDataDetector 兜底,MarkdownUI 自带 link 识别) |
| 图片 | ✅ | 网络图;message text 里有图片用 URL 时会加载 |
| 表格 | ✅ | macOS 13+ 要求,我们 26.4 满足 |
| 分隔线 `---` | ✅ | |
| 裸 URL 自动检测 | ❌ | MarkdownUI 只识别 `[label](url)` / `<url>` 格式,裸贴 `https://...` 不再变蓝。后续若有需求可在 MarkdownBody 内额外加 NSDataDetector pass。 |
