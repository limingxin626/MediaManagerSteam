# 2026-05-12 开发日志

## 移除 message text 中 #hashtag 自动解析为 tag 的逻辑

### 背景

历史上 `message.text` 中的 `#hashtag` 会被后端 `sync_tags_from_text()` 自动解析为 `Tag` 记录并关联到消息，前端 `MessageCard` 的快速添加标签按钮也通过把 `#tag` prepend 到 text 来"触发"这条解析。随着前端切到 `tag_ids` 显式接口、Android 端通过 SyncLog 走显式 tag 关联同步，文本解析路径变成了不一致的副作用来源（前端改 text 会自动改 tag，Android 改 text 不会，含糊不清）。统一改为：tag 关联只由 `tag_ids` 显式决定。

### 实现内容

**Frontend (vue)**

- `vue/src/components/MessageCard.vue` — `add-tag` event payload 由 `tagName: string` 改为 `tagId: number`,picker 透传 `tag.id`。
- `vue/src/views/Message.vue` — `handleQuickAddTag` 改写：不再 prepend `#tag` 到 `msg.text`,直接 PATCH `{ tag_ids: [...existingIds, tagId] }`,并通过返回的 `updated.tags` 同步本地状态。

**Backend**

- `backend/app/services/message_service.py` — 删除 `sync_tags_from_text()` 函数,清理 `re` / `Optional` / `Message` 等不再需要的 import。
- `backend/app/services/__init__.py` — 从导出列表移除 `sync_tags_from_text`。
- `backend/app/routers/message.py`:
  - `create_message`:不再回退到文本解析,`tag_ids` 为 `None` 时不建立任何 tag 关联（前端必须显式传)。
  - `create_message_from_client`:删除 `sync_tags_from_text` 调用。
  - `update_message`:删除"未传 tag_ids 时从 text 解析"的兼容分支。
  - `merge_messages`:合并 text 后不再重新解析 tag,target 自身的 tag 关联保留。
- `backend/app/routers/sync.py` — `_upsert_message` 创建和更新分支均删除 `sync_tags_from_text(merge=True)` 调用。Android push text 改动后,tag 关联不再受文本影响。
- `backend/scripts/import_flomo.py` — 把 hashtag 提取逻辑就地内联（只此一个外部场景需要从文本提取),不再依赖被删除的 service 函数。

### 注意

- Android 客户端目前 push 的 mutation payload 不携带 `tagIds`,本次改动后 Android 端对 text 的本地编辑不会再自动同步 tag 关联到后端。后续需要在 Android 侧补显式 tag 管理 UI 与对应 outbox 字段才能完整闭环。
- `import_flomo.py` 仍保留 `#hashtag` 文本提取,因为 flomo 原始数据是 markdown,这是导入场景的固有需求,不属于运行期解析。
