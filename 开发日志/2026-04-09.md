# 2026-04-09 开发日志

## 新建/编辑消息统一为弹出框

### 背景

原新建消息使用底部 MessageCompose 输入栏（单行 textarea，Enter 发送），输入长文本体验差，换行容易误触发送。编辑消息的弹框内嵌在每个 MessageCard 中，与新建逻辑高度重复。统一为共享的弹出对话框组件。

### 实现内容

**新建 `vue/src/components/MessageComposeDialog.vue`**
- 统一支持 `create` / `edit` 两种模式
- 全屏遮罩 + 居中卡片（max-w-3xl, h-[90vh]）
- 大 textarea 撑满可用空间，tag 自动补全正常工作
- 新建模式：附件选择（Electron 原生对话框 / 浏览器 input），文件预览
- 编辑模式：日期选择器，预填文本和日期
- Escape 关闭（优先关闭 tag 补全）
- 提交时自动 POST/emit，带 loading 状态

**改造 `vue/src/components/MessageCompose.vue`**
- 简化为只读触发器，点击整行打开新建弹框
- 移除所有输入逻辑、tag composable、文件处理、API 调用
- 仅 emit `open` 事件

**改造 `vue/src/components/MessageCard.vue`**
- 移除内嵌编辑对话框 HTML 和相关状态（editDialogVisible, editText, editDate, editTextareaRef, tag composable）
- 编辑按钮 emit `edit` 事件仅传 `id`（不再传 text/date）
- 移除 `allTags` prop 依赖

**改造 `vue/src/views/Message.vue`**
- 引入 MessageComposeDialog，管理弹框状态（dialogVisible, dialogMode, dialogMessageId, dialogInitialText, dialogInitialDate）
- `openCreateDialog` — MessageCompose 触发，打开 create 弹框
- `openEditDialog` — MessageCard edit 事件触发，从消息数据预填 text/date，打开 edit 弹框
- `onDialogCreated` — 新消息追加到列表，滚动到底部
- `onDialogUpdated` — PATCH API 更新消息，同步列表数据（含 tags）
