# 2026-05-29 无限滚动触发优化 + Media 桶加载优先级

## 背景

Vue 端 Message 页面无限滚动有"摸到边缘才加载"的轻微卡顿；对比 Android（Paging 3）几乎无痕。Media 时间轴在数据量大、跳转到远位置时，新位置的缩略图不会立即加载，似乎被旧请求挤住。

## 改动

### #1 Message 无限滚动改为"剩余 N 项"触发
- `vue/src/views/Message.vue`：删除 `topSentinel` / `bottomSentinel` 两个 1px 哨兵 div 和对应 ref；不再依赖 `rootMargin: 200px`。
- 新增 `PREFETCH_DISTANCE = 10`（pageSize 20 的一半）。`setupObservers` 观察列表中"距首/末 10 项"的真实消息节点，进入视口就预取。
- 新增 `rebindPrefetchTargets()`，在 `watch(messages)` 的 `nextTick` 中重新挂载到新的位置。
- 与滚动速度和卡片高度差异解耦，对齐 Android `prefetchDistance` 语义。

### #2 修复"回到最新"按钮的两个相关问题
- 现象：从记忆位置进入 Message 页时按钮一直显示；底部空白比正常时大；点击后空白收缩、按钮消失。
- 根因：滚动恢复路径无条件设 `isViewingHistory.value = true`（即使 forward 接口已经 `has_more=false`），sticky 按钮永远占着底部 ~48px。
- 修复：`isViewingHistory` 跟随 `forwardData.has_more`；forward 为空时显式置 `false`。

### #3 Media 虚拟网格按距离优先级排序
- `vue/src/composables/useVirtualGrid.ts`：新增 `enqueue(key, priority)`，每次入队后按"到视口中心距离"排序；`urgent` 走队首。
- `scheduleLoad`（dwell 后）和 `loadBucketNow`（立即）都改走 `enqueue`，去掉裸 `queue.push`。
- `scrollToBucket` / `scrollToDate` 跳转后立即调用 `loadBucketNow(key)`，并先手动同步 `scrollTop.value`，让距离排序立刻基于新位置生效。

### #4 提高并发上限
- `MAX_CONCURRENT` 2 → 6，与浏览器单 origin 图片并发对齐。LAN/本地场景下 SQLite 查询的瓶颈在 IO 调度而非带宽，2 太保守。

### #5 删除死代码
- `vue/src/views/MediaFeed.vue` 和 `router/index.ts` 里的 `/media/feed` 路由 —— 全仓库无导航入口，纯遗留。后端 `/media/feed` 接口暂留（待确认 Android 是否在用）。

## 未做（讨论后未实施）
- 跳转后取消所有正在跑的请求：abort 已发请求收益小（后端工作已经做了一半）、可能误伤新视口邻居、DateScrubber 频繁拖动时会反复 abort 服务端。如果优先级排序 + 并发 6 后仍能感到延迟，再考虑"只取消既不在视口也不在预取窗口的请求"。
- `useApi.ts` 加 `AbortController` 支持：上述方案的前置条件，等需要时再做。

## 验证
- `pnpm exec vue-tsc --noEmit` 通过
- commit: `0044da2`

# 2026-05-29（下午）Media 页面按天聚合

## 背景
Media 页面的桶粒度原本是「月」，月内媒体多时一个 bucket 高度数千 px，定位粗糙。改为按天后接近 Apple Photos / Google Photos 的节奏。

OpenSpec change: `media-timeline-day-grouping`。

## 改动

### 后端
- `backend/app/schemas/media.py:TimelineItem` 增加 `day: int`。
- `backend/app/routers/media.py:get_media_timeline` 增加 `day_col`，`group_by('year','month','day')`，`order_by year/month/day DESC`。

### 前端 `useVirtualGrid.ts`
- `TimelineEntry` / `BucketLayout` 增加 `day`。
- `bucketKey(y,m,d)` → `YYYY-MM-DD`。
- `monthStartIso` / `nextMonthStartIso` → `dayStartIso` / `nextDayStartIso`，后者用 `new Date(y,m-1,d+1)` 处理跨月跨年。
- `runLoad` 边界从 `monthStart` 改 `dayStart`。
- `findBucketByDate` 用 `y*10000+m*100+d` 三元整数比较查找首个 `<= target`。
- `scrollToDate` 简化为直接落到 `b.headerOffset`（不再按月内日期算 frac）。
- `currentDate` 精度到天。

### 前端 `Media.vue`
- 桶头文案 → `YYYY 年 M 月 D 日`。
- `timelineMin/MaxDate` 用 `last.day` / `first.day`。
- `handleDateScrubberJumpFinal` 的 key 拼接补上 `day` 段。

## 验证
- `pnpm exec vue-tsc --noEmit` 通过。
- 浏览器端联调待用户在 `/media` 页面手动验证 golden path / DateScrubber / 过滤器 / 删除最后一项 / Smart 模式不受影响。

## 未做
- 没有引入「月分组下嵌套天分组」的两级层次（提案明确 Non-Goal）。
- 没有把 `monthHeaderH` 改名为 `bucketHeaderH`，避免无意义 churn。

# 2026-05-29（晚2）视频预览「从已有图片添加」按 message 过滤

## 背景
MediaDetail 视频详情页的「从已有图片添加」picker 默认列全库图片，要找当前视频对应 message 下的图片得翻很久。改成只列出当前 video 关联的 message（可能多条）下的图片，取并集。

## 改动

### 后端
- `backend/app/routers/media.py:get_media`：新增 `message_ids: Optional[List[int]] = Query(None)`，在常规列表分支中按 `MessageMedia.message_id IN (...)` 过滤（与 `type=image`、cursor 分页正交）。原有单值 `message_id` 分支保留不动。
- `backend/app/routers/media.py:get_media_by_id`：返回类型由 `MediaResponse` 改为 `MediaDetailResponse`，额外查询 `MessageMedia` 填充 `messages=[{id}, ...]`，让前端拿到当前 media 的所有 message 关联。

### 前端
- `vue/src/composables/useApi.ts`：GET 的 `URLSearchParams` 拼接支持数组值（用 `append` 而非 `set`），匹配 FastAPI 对 `List[int]` Query 的 repeated key 解析。
- `vue/src/types.ts`：`Media` 类型扩展 `messages?: { id: number }[]`。
- `vue/src/components/MediaPickerModal.vue`：新增 `messageIds?: number[]` prop，非空时随 `/media` 请求带上。
- `vue/src/components/VideoPreviewStrip.vue`：新增 `messageIds?: number[]` prop，透传给 picker。
- `vue/src/views/MediaDetail.vue`：把 `mediaItem.messages.map(m => m.id)` 传给 `VideoPreviewStrip`。

## 验证
- `pnpm exec vue-tsc --noEmit` 通过。
- 浏览器端联调待手动验证：进入视频详情 → 打开 picker → 列表应只含与该视频共享 message 的图片。
