# 2026-06-10 Mac 消息 markdown 渲染:段内单换行修复

## 问题

用户反馈两件事:

1. **换行太严格**:很多视觉上分两行(行间无空行)的内容,渲染后被拼到一起 —— 看起来像一行
2. **跨段选择不通**:鼠标拖选时,只能选当前一段,相邻段落选不进去

## 根因

`MarkdownBody.swift` 走 swift-markdown-ui,严格 CommonMark 语义把段内单换行(soft break)当作空格,
所以"两行连在一起"。vue 端 `marked` 设置了 `breaks: true`,把单换行当成 `<br>`,二者行为不一致。

跨段选择不通是 swift-markdown-ui 的根本限制:每个 block(段落/标题/列表/引用/代码块)
独立渲染成 SwiftUI view,`.textSelection(.enabled)` 不能跨 view 选。

## 本期改动:仅修问题 1

`MarkdownBody.swift` 新增 `softBreakToHardBreak(_:)` 静态方法:
- CRLF / CR 归一化为 `\n`
- 围栏代码块(``` / ~~~)区间内原样保留,避免破坏代码
- 段内单换行追加两空格(CommonMark 硬换行语法),保证渲染为 `<br>`
- 已有硬换行标记(行尾两空格 / 反斜杠)或后续是空行 / 当前是空行的不动

`body` 内 `Markdown(text)` 改成 `Markdown(Self.softBreakToHardBreak(text))`,其它样式不动。

### 副作用:问题 2 部分缓解

修了问题 1 之后,原本"两行被错误连成一段"的内容现在变成同一段内的两行 —— 这种情况选择能跨行。
但真正不同段落(中间有空行)之间还是选不到一起。

## 没做的事:NSTextView 桥接重写

跟用户确认后**暂缓**。彻底解决问题 2 需要把 `MarkdownBody` 整个换成 `NSViewRepresentable` 包的
`NSTextView`(只读 + 可选择 + 自动高度),所有 block 走单一 `NSAttributedString`。

确认当前 markdown 渲染**没有任何业务交互**:
- 链接:仅系统默认 OpenURLAction(点了用默认浏览器打开),没有自定义拦截
- 任务列表 checkbox:仅展示,不可勾选
- 图片:不可点开
- 代码块:无复制按钮
- 表格:不可排序选行

所以 NSTextView 重写没有要保留的交互,代价主要在样式:
- 围栏代码块圆角底色 → 降级成方角(`NSAttributedString.backgroundColor` 是按字符画的)
- 引用块竖线 → 改成左缩进 + 灰字,或用前缀字符 hack
- 表格 / 任务列表 → 退化成纯文本

下次需要时再做。
