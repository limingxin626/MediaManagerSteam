# 2026-04-14 开发日志

## Android 底部导航栏改为悬浮样式

### 背景

原先使用 Material3 标准 `NavigationBar` + `NavigationBarItem`，固定在屏幕底部，占满整个宽度。改为现代悬浮药丸样式导航栏，提升视觉品质。

### 实现内容

**`MainActivity.kt` — 替换底部导航栏实现**

- 移除 Scaffold 的 `bottomBar` 参数，改用 `Box` 布局叠加内容和悬浮导航栏
- 内容区域 (`AppNavHost`) 填满整个屏幕，导航栏浮在内容上方
- 使用 `AnimatedVisibility` + `slideInVertically`/`slideOutVertically` 实现导航栏进出动画
- 导航栏距屏幕边缘 24dp 水平内边距 + 12dp 底部间距 + 系统导航栏安全区

**新增 `FloatingNavBar` 组件**

- `Row` 容器 + `RoundedCornerShape(28.dp)` 圆角胶囊形状
- `shadow(8.dp)` 阴影 + 半透明 `surfaceContainer` 背景 (95% 不透明度)
- 选中项：`primary` 色 15% 透明度圆角背景高亮 + `primary` 色图标/文字
- 未选中项：透明背景 + `onSurfaceVariant` 色图标/文字
- 移除波纹效果 (`indication = null`)，保持简洁
- 图标 22dp，标签使用 `labelSmall` 字体

### Android 文件变更

- `android/.../MainActivity.kt`：新增多个 import；Scaffold 移除 bottomBar 改用 Box 叠加；新增 `FloatingNavBar` composable

## 后端与前端开发服务改为仅本机访问

### 问题背景

调试 Electron 时发现两个网络侧问题：

- FastAPI 后端监听 `0.0.0.0`，并且 CORS 对所有来源放开
- Vite 开发服务器显式配置了局域网 `hmr.host`，导致 Electron 总是尝试连接 `192.168.31.146:5173` 的热更新 WebSocket

当前需求是只允许本机访问，避免服务继续暴露到局域网，也避免开发环境里出现无意义的局域网 HMR 连接报错。

### 收口实现

#### 后端 FastAPI 收口为 localhost

- 在 `backend/app/config.py` 新增 `HOST` 配置，默认值为 `127.0.0.1`
- 新增 `ALLOWED_ORIGINS` 白名单，仅允许 `localhost` / `127.0.0.1` 及本地常用端口来源访问
- `backend/api.py` 启动入口改为使用 `config.HOST`，不再固定监听 `0.0.0.0`
- `backend/app/__init__.py` 的 CORS 从 `allow_origins=["*"]` 改为使用配置白名单

#### Vite 开发服务器收口为 localhost

- `vue/vite.config.ts` 中 `server.host` 从 `0.0.0.0` 改为 `127.0.0.1`
- 删除固定的 `server.hmr.host = 192.168.31.146`
- Electron 开发模式下继续加载 `http://localhost:5173`，但不再尝试连接局域网 IP 的 HMR WebSocket

### 结果

- FastAPI 仅接受本机访问
- 浏览器 / Electron 前端仅从本机访问 Vite dev server
- Vite 热更新 WebSocket 不再指向局域网地址，开发控制台中的 `ws://192.168.31.146:5173` 连接超时问题消失

### 涉及文件

- `backend/app/config.py`：新增 `HOST` 和 `ALLOWED_ORIGINS` 配置，默认只允许本机访问
- `backend/app/__init__.py`：CORS 从全开放改为本机来源白名单
- `backend/api.py`：Uvicorn 启动 host 改为读取 `config.HOST`
- `vue/vite.config.ts`：开发服务器 host 改为 `127.0.0.1`，移除局域网 HMR host
