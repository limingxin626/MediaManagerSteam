# 2026-04-08 开发日志

## 搜索交互改为手动触发

### 背景

原 `SearchInput` 组件使用 300ms debounce，每次输入都会自动触发搜索请求，体验上不够克制。

### 实现内容

**修改 `vue/src/components/SearchInput.vue`**
- 移除 debounce 逻辑和 `onUnmounted` 清理
- 输入时仅更新 `modelValue`，不触发 `search` 事件
- 新增回车键监听（`@keydown.enter`）触发搜索
- 新增"搜索"按钮，点击触发搜索

### 文件变更清单

| 文件 | 变更 |
|------|------|
| `vue/src/components/SearchInput.vue` | 改为按回车/点击按钮触发搜索 |

## 为 Backend 创建独立 MCP Server

### 背景

为了让各种 AI agent（如 Claude Code）能通过 MCP 协议直接操作 MediaManager 后端，创建一个独立轻量的 MCP 服务器。该服务器通过 HTTP 调用后端 FastAPI API，不直接依赖后端代码，可放在任意 agent 目录下使用。

### 实现内容

**新建 `backend/mcp_server.py`** — 独立单文件 MCP 服务器，仅依赖 `mcp` + `httpx`。

提供 5 个 tools：
- `write_message` — 创建消息（支持文本、Actor、文件路径）
- `search_messages` — 搜索消息（支持文本/Actor/Tag/收藏过滤，游标分页）
- `read_message` — 读取单条消息详情
- `list_actors` — 列出所有 Actor（支持名称搜索）
- `list_tags` — 列出所有 Tag（支持名称搜索）

通过环境变量 `API_BASE_URL`（默认 `http://localhost:8002`）配置后端地址。

### 文件变更清单

| 文件 | 变更 |
|------|------|
| `backend/mcp_server.py` | 新建：独立 MCP 服务器，5 个 tools |
| `backend/pyproject.toml` | 添加 `mcp>=1.0.0` 依赖 |
