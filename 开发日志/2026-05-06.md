# 2026-05-06 开发日志

## Media 页面长列表卡顿优化（真正的虚拟滚动）

### 背景

媒体数量达到十万级别时，Media 页面上下滚动明显卡顿，即使滚到已加载过的区域也不流畅。

### 原因

`useVirtualGrid` 此前只对**数据加载**做了虚拟化（仅可视 bucket 触发分页请求），但模板里仍把 `buckets × b.count` 全量展开成绝对定位的占位 `<div>`，10 万条 = 10 万个 DOM 节点，外加每个 cell 多次调用响应式 `vg.loadedItem(...)`，主线程被 layout / patch 拖垮。

### 实现内容

**`vue/src/composables/useVirtualGrid.ts`**

- 新增 `visibleCells` computed：根据 `scrollTop / viewportH / RENDER_OVERSCAN_PX(400)`，对每个 bucket 计算与视口相交的行区间，仅产出该区间内的 cell 的 `{ bucket, idx, top, left, size }`。
- 导出 `visibleCells`，新增 `RENDER_OVERSCAN_PX` 常量（与数据层 `PREFETCH_PX=800` 区分：渲染层窗口比预取窗口窄）。

**`vue/src/views/Media.vue`**

- 月份 header 改用 `vg.visibleBuckets.value` 渲染（仅渲染视口附近的月份标题）。
- cells 改为遍历 `vg.visibleCells.value`，`v-for` 节点数从 `O(总媒体数)` 降到 `O(视口内 cell 数)`，约 20–40 个。
- 抽出 `vue/src/components/MediaCell.vue` 承载 thumb / 视频角标 / duration / 收藏按钮，避免模板里对同一个 `loadedItem` 反复调用响应式 getter。

### 涉及文件

- `vue/src/composables/useVirtualGrid.ts`
- `vue/src/views/Media.vue`
- `vue/src/components/MediaCell.vue`（新增）
