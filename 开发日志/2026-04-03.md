# 2026-04-03 开发日志

## 实现前后端双向同步架构

### 背景

之前的同步方案存在以下问题：
- 全量同步不可扩展（每次拉取全部数据）
- 后端没有 `POST /api/sync/apply` 端点（Android push 完全不工作）
- 缺少删除追踪（硬删除后客户端无感知）
- 无后台同步、无网络检测、无实时推送

### 实现内容

#### 后端

**新增 `SyncLog` 模型** (`backend/app/models/__init__.py`)
- 记录 `MESSAGE | ACTOR | MEDIA | TAG` 的 `UPSERT | DELETE` 操作
- 复合索引 `(timestamp, id)` + `(entity_type, entity_id)`
- 保留期 90 天

**新增 `sync_log_service.py`**
- SQLAlchemy `after_flush` 事件监听器：与业务数据在同一事务中原子写入
- 使用 thread-local 防止递归 flush
- 进程内 SSE 事件总线（`asyncio.Queue` 列表）
- `after_commit` 后广播通知给所有 SSE 订阅者

**新增 `routers/sync.py`** — 三个端点：
1. `GET /sync/changes?since={ISO}&limit=500`
   - 增量拉取，返回去重后的变更列表
   - UPSERT 附带完整实体快照（MESSAGE 含嵌套 media_items + tags）
   - since 为空/超过 90 天 → HTTP 410，客户端回退全量同步
2. `POST /api/sync/apply`
   - 按序处理 Android 推送的变更
   - Last-write-wins by `updated_at`，逐条 try/catch 不中断批次
   - camelCase → snake_case 字段映射
3. `GET /sync/events`
   - SSE 实时通知（仅携带元数据，不含完整数据）
   - 30 秒心跳保活

**修改 `app/__init__.py`**
- 启动时注册 SyncLog 监听器
- `Base.metadata.create_all(checkfirst=True)` 自动建表

#### Android

**新增依赖** (`build.gradle.kts`)
- `okhttp3:okhttp:4.12.0` + `okhttp3:okhttp-sse:4.12.0`（SSE）
- `androidx.work:work-runtime-ktx:2.9.1`（后台同步）

**扩展 `SyncService.kt`**
- 新增 `getChanges(since, limit)` 接口，返回 `Response<RemoteChangesResponse>`

**新增 `SyncModels`**
- `RemoteChangesResponse` / `RemoteChangeItem`
- `SyncResult.NeedFullSync`（410 响应时使用）

**`MessageRepository.syncIncremental(since)`**
- 增量拉取，处理 UPSERT（MESSAGE/ACTOR/MEDIA/TAG）和 DELETE
- has_more=true 时循环拉取
- 410 → 返回 `NeedFullSync`，调用方回退全量同步
- Gson 反序列化 `Map<String, Any?>` → 类型安全的 data class

**新建 `NetworkMonitor.kt`**
- `ConnectivityManager.NetworkCallback` 监听 WiFi 状态
- 暴露 `StateFlow<Boolean>` 供订阅

**新建 `SyncWorker.kt`**
- `CoroutineWorker`：push outbox → pull 增量（410 → 全量）
- `schedulePeriodicSync()`：每 15 分钟，WiFi constraint
- `scheduleImmediateSync()`：网络恢复时一次性触发
- `SharedPreferences` 存储 `last_sync_time`

**新建 `SseListener.kt`**
- OkHttp `EventSource` 连接 `/sync/events`
- 500ms debounce 触发增量同步
- 指数退避重连（2s → 30s）

**`MainActivity.kt`**
- 启动时调度定期同步
- 监听网络恢复触发即时同步

**`DatabaseManager.kt`**
- 注入 `NetworkMonitor`

#### Vue

**新建 `useSyncEvents.ts`**
- `EventSource` 连接 `/sync/events`
- 自动重连（指数退避）
- 回调接收 `(entityType, entityId, operation)`
