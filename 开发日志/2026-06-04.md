# 2026-06-04 Mac app 本地数据库化(第一期)

## 背景

之前 Mac app(`MyNote/`)是 SwiftUI UI + HTTP 调本地 Python backend 的薄壳实现,跟 Vue/Electron 共用一套 backend。新定位是 Mac 作为主力端、Android 作为辅助、不再有 Web/Windows。Mac 不再需要依赖 Python 进程 —— 可以拿到原生分发(单 .app)、更快冷启动、QuickLook/AVKit/Spotlight 等系统集成,并节省 uvicorn 常驻进程的资源占用。

多端不必功能完全一致,只要 **数据库 schema 一致**(同一份 `db.sqlite3` + `data/thumbs/` + `uploads/` 布局),Mac 端完全本机直读 SQLite,不经过 backend。

本期只做 **媒体网格的本地化只读**,不涉及多设备同步、不涉及写操作。

### #4 设计决策(详见 `design.md`)

- **GRDB.swift v6.x** 而非 SwiftData(后者隐藏字段无法对齐 backend schema)
- 只读连接 `Configuration.readonly = true`,避免污染 backend / Android sync 状态
- 复合游标 `(created_at, id)` 而非 backend 的 `(created_at, position)` —— 本地查的是裸 `Media` 表,语义不同
- `NSImage` + `NSCache` 替代 `AsyncImage`,解决滚动重复解码 webp 卡顿
- Mac 第一期 **完全只读**,UI 隐藏 starred toggle 等写入入口;写路径仍走 HTTP(短期)
- QuickLook 推迟到下一期(需要 NSResponder chain 控制,作为独立 change 更合适)
