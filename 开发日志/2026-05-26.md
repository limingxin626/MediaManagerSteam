# 2026-05-26 开发日志

## 笔记全文检索 (backend FTS5)

### 背景

笔记数量增长后，MCP `search_messages` 直接返回完整 message（含全部 media URLs + actor + tags 对象），单条 1-3KB，搜 10 条就把 Claude Code 上下文塞满，且和 `read_message` 职责重合。原检索后端用 `Message.text.ilike('%q%')`，没有 ranking，没有命中片段。

### 方案

后端做 SQLite FTS5 全文索引，新增 `/messages/search` 精简端点。MCP `search_messages` 切到新端点，只返回 snippet + 元数据；详细内容由 `read_message` 提供。

### 实现

**FTS 表 (migration `a7b8c9d0e1f2`)**

FTS5 external content 虚表，rowid 指向 `message.id`，不复制正文：

```sql
CREATE VIRTUAL TABLE message_fts USING fts5(
    text, content='message', content_rowid='id',
    tokenize='unicode61 remove_diacritics 2'
);
```

3 个 trigger（`message_fts_ai/ad/au`）跟 `message.text` 的 INSERT/DELETE/UPDATE OF text 同步。upgrade 时一次性回填现有 62 行。SQLAlchemy 的 `after_flush` SyncLog 监听器和 SQL trigger 是两套独立机制，互不冲突。

**Endpoint `GET /messages/search`**

- `q`: 必填关键词，多 token 用 `_escape_fts_query` 包成 `"a" "b"` 短语（隐式 AND）
- 排序 `bm25(message_fts) ASC`，相同 rank 时 `id DESC` 兜底
- `snippet(message_fts, 0, '«', '»', '…', 16)` 生成 16 token 窗口的命中片段
- 过滤参数 `actor_id` / `issue_id` / `tag_id` / `starred` 复用现有过滤语义（`actor_id=0` = NULL）
- 游标 `{rank}|{id}`，配合 `bm25 > c_rank OR (bm25 = c_rank AND id < c_id)`

**中文 fallback**

unicode61 把连续中文当一整个 token —— 笔记里没空格分隔的「工作」搜不到（FTS 0 命中），但「会议」「简报」（在标题里独立成段）能命中。trigram 要 ≥ 3 字符不适用 2 字搜索。

折中：FTS 0 命中且无游标（首页）时降级走 `LIKE '%...%'`，Python 端手动生成 snippet（命中位置前后 30 字 + `«»` 高亮）。fallback 路径不分页（`has_more=false`，避免 rank=0 游标和 FTS 路径冲突）。

**Schema 精简**

新增 `MessageSearchItem`：仅 `id` / `created_at` / `snippet` / `actor_id+name` / `issue_id+title` / `tags`（name list）/ `media_count` / `starred`。完整 media URLs、tag category 由 `read_message` 提供。

**MCP**

`search_messages` 参数 `query_text` 改必填，去掉「不传 query 就列所有」用法（那个用 `list_issues` / `list_tags` 替代）。调 `/messages/search`，透传游标。

### 验证

```
GET /messages/search?q=简报         → 17 命中, bm25 排序，cursor 分页 OK
GET /messages/search?q=工作         → LIKE fallback, snippet 含 «工作» 高亮
GET /messages/search?q=工作 会议    → FTS 0 命中 → LIKE 拼成「工作会议」兜底
GET /messages/search?q=             → 空查询返回空数组（不报 400）
```

### 没做的

- pinyin/同义词
- trigram tokenizer（2 字中文搜索仍要 LIKE）
- snippet 长度可调（先固定 16 token）
- FTS 重建命令（rebuild 通过 `INSERT INTO message_fts(message_fts) VALUES('rebuild')`，需要时手动跑）

## 媒体智能查询（CLIP 本地）

### 背景

Media 页只能按 tag/actor/类型/收藏过滤。新增两件事：
1. 给图片/视频自动推荐 tag（图→文相关度）
2. 找视觉相似的媒体；顺带做文本→图片语义搜索

要求：完全本地、不出网、不影响上传管线性能。手动触发即可。

### 方案

CLIP ViT-B/32 onnx + onnxruntime CPU。embedding 512 维 float16 存 SQLite BLOB（≈1KB/条），库规模万级以内暴力 cosine 足够，不上 faiss。

新表 `media_embedding(media_id PK FK→media, model, vector BLOB, created_at)`，迁移 `b8c9d0e1f2a3`。

### 后端

- `app/services/clip_service.py` — 懒加载 visual.onnx / textual.onnx / tokenizer.json，模型放 `{DATA_ROOT}/models/clip/`。`encode_image / encode_text / vector_to_bytes / bytes_to_vector`。文本 encode 用 `lru_cache(2048)`，候选词重复查询零开销。
- `app/services/smart_tag_service.py` — `ensure_embedding`（缓存命中直接返回）、`suggest_tags`（候选 = 库内 Top-200 tag，按 cosine 排序返回 Top-K）、`find_similar`、`search_by_text`、`rebuild_embeddings`、`apply_tags`（追加不替换）。`find_similar` / `search_by_text` 一次性把所有 embedding stack 成 (N, 512) 矩阵做矩阵乘，远比逐条点积快。
- `app/routers/smart.py` — `/smart/status`、`/smart/tags/suggest`、`/smart/tags/apply`、`/smart/similar/{id}`、`/smart/search?q=`、`/smart/embeddings/rebuild`。模型不可用时所有 inference 端点 503，`/status` 始终返回。

embedding 不进 SyncLog（Android 不消费）。视频只用现成的 webp 缩略图，避免重复 ffmpeg。

### 前端

- `TagSuggestDrawer.vue` — 右侧抽屉显示 Top-12 建议（带 cosine 分数条），多选→`/smart/tags/apply` 追加 tag。
- `MediaPreview.vue` — 工具栏加两个图标按钮：「智能 tag」打开抽屉；「找相似」emit `find-similar` 给父组件并关闭预览。
- `Media.vue` — 顶栏加搜索输入框；输入回车走 `/smart/search`，预览页「找相似」走 `/smart/similar/{id}`，结果一次性 50 张平铺渲染。smart 模式激活时虚拟滚动 + DateScrubber 隐藏（`v-show` + `v-if`）。右上 × 按钮退出。

### 验证

1. 装依赖：onnxruntime / numpy / pillow / tokenizers 加到 pyproject 主依赖。
2. `alembic upgrade head` → `media_embedding` 建表。
3. 放置模型到 `{DATA_ROOT}/models/clip/`（README 有说明）。
4. `POST /smart/embeddings/rebuild` 预热全库。
5. 前端：预览页「智能 tag」→ Top-12 建议 → 勾选应用 → 媒体 tag 更新。预览页「找相似」→ 切到相似 grid。顶栏搜索 "sunset" / "猫" → 语义结果。

### 没做的

- 视频中间帧采样（仅首帧）
- 向量索引（faiss/sqlite-vss）
- 上传管线自动 embedding（保持完全手动触发）
- 内置候选词表，仅用 `Tag.name`
- Android 端 smart 路由（embedding 不进 SyncLog）
