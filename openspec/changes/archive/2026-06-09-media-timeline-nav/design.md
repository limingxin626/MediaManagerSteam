## Context

Media 页面当前使用无限滚动加载媒体 grid（按 `created_at DESC` 排序）。用户无法快速跳转到特定时间段，需要持续滚动才能到达较早的内容。Message 页面已有类似的日期跳转功能（`/messages/dates` 接口 + v-calendar），但 Media 页面尚未实现。

现有 cursor 机制支持 `direction=forward`，可复用于时间轴跳转。

## Goals / Non-Goals

**Goals:**
- 提供按月聚合的时间轴导航，快速跳转到任意月份
- 滚动时同步更新时间轴上的当前位置
- 在 grid 中插入月份分隔头，提供时间上下文

**Non-Goals:**
- 不做按天粒度的时间轴（数据量过大，月粒度即可）
- 不做缩略图预览（hover 预览时间轴上的内容）
- 不做时间范围筛选（只做跳转，不做过滤）

## Decisions

### 1. 时间轴交互方式：右侧固定竖条

采用右侧固定的竖条式时间轴，类似 Telegram 和 Windows 照片。时间轴始终可见，显示年月标签。

**替代方案**：顶部水平时间轴 — 占用宝贵的垂直空间，移动端体验差。

### 2. 跳转实现：重置 items + 从目标 cursor 加载

点击月份时，构造该月第一天的 cursor（`"YYYY-MM-01T00:00:00|0"`），清空现有 items，以该 cursor 为起点重新加载。这与现有 cursor 分页机制完全一致，无需新的后端分页逻辑。

**替代方案**：预加载所有数据并在前端定位 — 内存消耗过大，不可行。

### 3. 滚动位置同步：从可视区域内 item 推断当前月份

通过 IntersectionObserver 或滚动事件检测当前可视区域顶部的 item，读取其 `created_at` 来更新时间轴高亮。月份分隔头也可作为观测目标。

### 4. 后端聚合接口：`GET /media/timeline`

返回 `[{year, month, count}]` 数组，按时间降序。使用 SQLite `strftime` 按月分组聚合。需接受与 `/media` 相同的 `type`、`starred` 过滤参数以保持一致。

### 5. 月份分隔头插入方式：前端在 items 间插入虚拟元素

在渲染时比较相邻 item 的月份，不同月份之间插入日期分隔头。不修改后端返回数据结构。

## Risks / Trade-offs

- [跳转后无法回到之前位置] → 可记录跳转前的 scroll 位置和 cursor，提供"返回"功能（但作为 v1 不实现，用户可通过再次点击时间轴返回）
- [大量月份时时间轴过长] → 年份折叠（只显示年标签，展开后显示月份）。v1 先全部展开，数据量不大时可接受
- [过滤条件变化时需刷新时间轴数据] → 过滤参数变化时重新请求 `/media/timeline`
