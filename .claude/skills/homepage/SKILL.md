## 步骤

1. 备份当前 `homepage.md` 到 `homepage_history/homepage_YYYY-MM-DD.md`（当天已有则跳过）
2. 读取当前 `homepage.md` 全部内容
3. 结合今日简报（`reports/briefing_YYYY-MM-DD.md`）或最新数据，增量更新 `homepage.md`

如果今日简报不存在，可以调用 `get_briefing()` 获取最新数据作为更新依据。

## 更新原则

- 新发现的待办 → 加入"需要行动"
- 已回复/已处理的事项 → 移到"近期完成"并标注完成日期
- 持续跟进的事项 → 更新"跟进中"的状态描述
- 定期会议/周期性事项 → 加入"周期性备忘"
- **保留用户手动添加的条目** — 不要删除不在日报中出现的条目
- "近期完成"中超过 7 天的条目可以清理

## Homepage 格式

```markdown
# 工作仪表盘
最后更新: YYYY-MM-DD

## 需要行动 (Action Required)
- [ ] 事项描述 — 来源/备注

## 跟进中 (Tracking)
- 事项描述 — 当前状态

## 周期性备忘
- 会议/定期事项

## 近期完成
- ✅ 事项描述 — 完成日期
```

## 格式规则

- 更新日期写入"最后更新"行
- 每个条目简洁一行，必要时用 `—` 分隔描述和备注
- "需要行动"使用 `- [ ]` 复选框格式
- "近期完成"使用 `✅` 前缀
