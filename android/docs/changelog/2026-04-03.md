# 2026-04-03 开发日志

## 功能：改造 ActorListScreen 为 Actor 分组页面

### 需求

将演员列表页面改造为仿 HomeScreen 风格的分组页面：
- 顶部显示"全部"（含总消息数）
- 下方每个 actor 按消息数量降序排列
- 点击进入对应 actor 的消息列表

### 修改文件

1. **`MessageDao.kt`** — 添加 4 个 actor 查询方法：
   - `getMessagesByActorPaged(actorId)` - PagingSource
   - `searchMessagesByActorPaged(actorId, query)` - 带搜索的 PagingSource
   - `getMessageCountByActor(actorId)` - 消息数量统计
   - `getLastMessageByActor(actorId)` - 最新消息预览

2. **`MessageRepository.kt`** — 暴露 3 个 actor 相关方法：
   - `getMessagesByActorPaged(actorId, query)`
   - `getMessageCountByActor(actorId)`
   - `getLastMessageByActor(actorId)`

3. **`MessageViewModel.kt`** — 添加 actorId 过滤支持：
   - 新增 `_actorId: MutableStateFlow<Long?>`
   - `messagesPaged` 改为三路 combine（query + tagId + actorId）
   - 优先级：actorId > tagId > 全部
   - 新增 `setActorId(actorId: Long?)` 方法

4. **`ActorViewModel.kt`** — 完全重写（参照 HomeViewModel 模式）：
   - 新增 `ActorGroupItem` 数据类
   - `groups: StateFlow<List<ActorGroupItem>>` 替代原 actors 列表
   - 收集 actorRepository.getAllActors()，对每个 actor 查询消息数和最新消息
   - 按消息数降序排列，"全部"置顶

5. **`ActorListScreen.kt`** — 完全重写（参照 HomeScreen 风格）：
   - 移除搜索、筛选、FAB、底部筛选面板
   - LazyColumn 展示 groups，支持圆形头像（Coil 加载或首字母）
   - 函数签名：`onNavigateToMessages: (Long?) -> Unit`

6. **`Navigation.kt`** — 添加路由：
   - `MESSAGE_LIST_BY_ACTOR = "messages_by_actor?actorId={actorId}"`
   - `messageListByActor(actorId)` builder 方法

7. **`MainActivity.kt`** — 多处更新：
   - `showBottomBar` 加入 `Routes.ACTOR_LIST`，移除 `Routes.TAG_LIST`
   - `AppDestinations` 加入 `ACTORS` Tab（用 Person 图标），移除 `TAGS`
   - ACTOR_LIST composable 改为调用新签名的 ActorListScreen
   - 注册 `MESSAGE_LIST_BY_ACTOR` 路由，通过 `messageViewModel.setActorId()` 传参

### 注意

项目中存在预先存在的编译错误（`MessageRepository.kt` 中 `RemoteMessage` 等未解析引用，
`SettingsViewModel.kt` 中 when 表达式不全），这些错误在本次修改前已存在，不影响本次功能。
