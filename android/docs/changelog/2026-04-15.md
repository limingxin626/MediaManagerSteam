# 2026-04-15 开发日志

## 修复：媒体预览界面控件显隐 + 拖拽惯性

### 问题

1. **控件显隐跨页不持续**：缩略图条在滑动切换页面时状态不一致——当前页隐藏后切到下一页又显示
2. **视频控件无法隐藏**：视频播放器的控件（播放按钮、进度条）始终显示，隐藏操作无效
3. **拖拽无惯性**：放大后拖拽松手立即停止，缺乏动量感

### 修改文件

1. **`VideoPlayer.kt`** — TelegramVideoPlayer 控件状态改为受控模式：
   - 新增 `controlsVisible: Boolean` 参数，移除内部 `var controlsVisible` 状态
   - `toggleControls()` 和 `scheduleHideControls()` 改为通过 `onControlsVisibilityChanged` 回调请求外部变更
   - 新增 `LaunchedEffect(controlsVisible)` 在外部状态变为 true 时启动自动隐藏计时
   - 添加 `VelocityTracker` + `splineBasedDecay` 的 fling 惯性动画

2. **`MediaViewerScreen.kt`** — 传入 controlsVisible 参数：
   - `TelegramVideoPlayer` 调用处增加 `controlsVisible = controlsVisible` 参数

3. **`ZoomableContainer.kt`** — 添加 fling 惯性动画：
   - 引入 `VelocityTracker`，在 pan 时追踪手指速度
   - 手势结束时使用 `animateDecay(velocity, splineBasedDecay)` 启动衰减动画
   - 通过 `Animatable.updateBounds()` 设置边界，动画到达边界时自然停止

### 技术要点

- 使用 Compose 的状态提升模式（state hoisting）实现视频控件双向同步
- `splineBasedDecay` 是 Android 平台原生 fling 衰减曲线，与系统滚动行为一致
- `Animatable.updateBounds()` 在 fling 前设置、fling 后清除，确保边界约束仅在惯性滑动期间生效
