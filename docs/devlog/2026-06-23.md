## feat(backend+vue): 磁盘扫描视图(Scan / `fs_entry`)—— 追踪未注册/快速变化的媒体文件

### 背景

现有 `/media` 视图的数据源是 `media` 表 —— 已去重(`file_hash` UNIQUE)、已 import 的**逻辑资产库**。但它有两个盲区:

1. 看不到**还没 import 成 Media 行**的文件;
2. 看不到磁盘上**重复**的文件(同内容两个路径,`Media` 去重模型只保留一行)。

需求:扫描所有注册 repository 目录下的媒体文件,以 grid 展示(按时间/大小/名称排序),用来**追踪可能快速变化的目录**,且要呈现**丰富 metadata**(分辨率、GPS 位置、相机、HDR、编码、时长等)。规模十万量级。

这是一个与 `Media` 正交的「**磁盘物理真相**」视图。关键语义差异:`fs_entry` **不去重**,唯一约束只有 `(repo_id, rel_path)`(路径唯一,内容可重复)—— 这正是它能看到重复/未注册文件的原因。两张独立表,`fs_entry` 永不污染资产库。

### 设计要点

**两阶段填充**(关键:十万文件不能在一次请求里全读完):
1. **rescan(同步,只 `os.stat`)** —— 秒级,填 repo_id/rel_path/size/mtime/media_type/mime/media_id。grid 立刻能按 size/mtime/name 排序、翻页。
2. **后台 worker(逐个,读文件)** —— ffprobe/EXIF 抽 metadata + HDR + 生成缩略图,慢慢填回。grid 先看占位 spinner,好一个出一个。

**缩略图复用**:rescan 时按**精确 `(repo_id, rel_path)` 匹配** `media` 表(不算 hash)。命中 → `fs_entry.media_id` 记下,worker 直接从 Media 行搬 metadata(不读文件)、复用现成 `/data/thumbs/{media_id}.webp`(置 `reused`);未命中 → worker 读文件 + 生成 `/data/scan_thumbs/{fs_entry_id}.webp`。这省掉了对已注册文件的整整一遍重复抽取。

**mark-and-sweep**:每次扫描用 `run_token`(datetime)写到每行 `scanned_at`,走完删掉 `scanned_at < run_token` 的行(= 磁盘上已消失的文件),并清理其自有 scan_thumb(reused 行不动 Media 缩略图)。

## feat(vue): Scan 全屏预览(复用 MediaPreview 精简模式) + fix 关闭弹窗报错

### 背景

Scan 页初版点 cell 只开 `ScanDetailModal`(纯 metadata)。需求对齐 `/media`:支持**全屏打开 + 左右翻看**。

### 改动

1. **复用而非新造** —— `MediaPreview.vue` 已具备全屏遮罩 / 左右箭头 + 键盘导航 / 缩略图条 / slide 过渡 / 文件失败回退缩略图,且 `FsEntry` 自带它读的全部字段(`id`/`mime_type`/`file_url`/`thumb_url`/`local_*_path`/`width`/`height`/`duration_ms`)。问题只在它的工具栏全是 Media 资产操作(收藏/删除/旋转/替换/智能 tag/找相似/详情页),用 `fs_entry.id` 打 `/media` 接口是错的(且 `media_id` 可能为 null)。

2. **`MediaPreview.vue`** —— 加 `minimal?: boolean` prop:`v-if="!minimal"` 包住整组资产操作按钮 + 底部 tag 编辑器;精简模式只留 导航 + 预览 + 打开文件位置 + 关闭,并新增一个「详情」按钮 emit `info` 事件(宿主自定义,Scan 用它开 `ScanDetailModal`)。默认 `false`,`/media` 行为完全不变。

3. **`Scan.vue`** —— cell 点击改为 `openPreview(idx)` 开全屏(`items` 数组直接传给 MediaPreview,索引复用);`@navigate-next` 翻到已加载末尾时追加下一页(`load()`);`@info` 开 `ScanDetailModal` 看完整 EXIF/GPS/编码。从 `useInfiniteScroll` 多取 `load`/`hasMore`。

4. **fix:`ScanDetailModal.vue` 关闭报错** —— 原 `DialogPanel` 用 `v-if="item"`,而外层 `TransitionRoot :show="!!item"`。关闭时父级置 `item=null` **同时**触发 `:show→false`(HeadlessUI 要播离场动画、保留 DOM)和 `v-if→false`(Vue 立刻销毁子树),两者争抢同一段 DOM 把 vnode 树搞坏 → `Cannot read properties of null (reading 'type')`。改为缓存最后一次非空 item 到本地 `display` ref(只增不清),离场动画期间继续渲染旧内容,挂载/卸载交给 `:show`。

### 验证

`pnpm vue-tsc --noEmit` 通过。Scan 页点 cell 开全屏、左右翻看、关闭无报错;翻到末尾自动加载下一页;「详情」按钮开 metadata modal。
