# 2026-04-29 开发日志

## 主页 Dashboard + Todo 看板

**动机：** 旧主页（2026-04-27 加的 Markdown 仪表板）需要手动编辑 Markdown 文本来更新 todo 状态，太笨重。改造成结构化 dashboard，todo 走数据库 + 拖拽看板。

**改动：**

后端：
- 新建 `Todo` 表（`title / status / position / created_at / updated_at / completed_at`，独立模块 `app/models/todo.py`）+ alembic 迁移 `4d40c4acab1e_add_todo`。
- `app/services/todo_service.py` 处理列内/跨列移动的位置重排：把目标列（不含当前 todo）算成新列表，插入指定位置后整列重写 0..N-1 的连续 position；跨列时源列也压紧。已完成列只展示最近 20 条（`DONE_LIMIT`）。
- `app/routers/todos.py` 五个端点：`GET /todos`（看板分组）/ `POST /todos`（新增）/ `PATCH /todos/{id}`（改标题）/ `PATCH /todos/{id}/move`（移动）/ `DELETE /todos/{id}`。
- `app/routers/dashboard.py` 完全替换：删除 markdown 文件 GET/PUT，仅保留 `GET /api/dashboard/stats` 聚合 4 个指标（消息数 / 媒体数 / 本月新增媒体 / 进行中 todo）。
- `app/config.py` 同步删除 `DASHBOARD_DEFAULT_CONTENT` 常量与 `get_dashboard_md_path()` 方法。

前端：
- 安装 `vuedraggable@^4`（基于 SortableJS）。
- `vue/src/types.ts` 新增 `Todo / TodoStatus / TodoBoard / DashboardStats` 类型。
- `vue/src/components/dashboard/` 四个组件：
  - `TodoCard.vue` — 单卡，双击编辑、hover 显示删除 ✕、done 状态删除线。
  - `TodoColumn.vue` — 单列，包含 vuedraggable list；pending 列底部多一个新增输入框。
  - `TodoBoard.vue` — 三列容器，处理 `vuedraggable` 的 `change` 事件 → 调 `PATCH /todos/{id}/move`，失败 toast + reload 回滚。
  - `RecentMedia.vue` — `/media?limit=12`，3-6 列响应式宫格。
  - `StatsCard.vue` — 调 `/api/dashboard/stats`，4 个数字卡 2x2 网格。
- `vue/src/views/Dashboard.vue` 完全重写为 widget 容器（之前 135 行 markdown 编辑器现在 24 行装配）。

1. 看板三列正确显示，最近媒体九宫格 + 4 个统计卡可见。
