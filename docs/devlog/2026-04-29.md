# 2026-04-29 开发日志

## 主页 Dashboard + Todo 看板

**动机：** 旧主页（2026-04-27 加的 Markdown 仪表板）需要手动编辑 Markdown 文本来更新 todo 状态，太笨重。改造成结构化 dashboard，todo 走数据库 + 拖拽看板。

**改动：**

后端：
- 新建 `Todo` 表（`title / status / position / created_at / updated_at / completed_at`，独立模块 `app/models/todo.py`）+ alembic 迁移 `4d40c4acab1e_add_todo`。
- `app/services/todo_service.py` 处理列内/跨列移动的位置重排：把目标列（不含当前 todo）算成新列表，插入指定位置后整列重写 0..N-1 的连续 position；跨列时源列也压紧。已完成列只展示最近 20 条（`DONE_LIMIT`）。
- `app/routers/todos.py` 五个端点：`GET /todos`（看板分组）/ `POST /todos`（新增）/ `PATCH /todos/{id}`（改标题）/ `PATCH /todos/{id}/move`（移动）/ `DELETE /todos/{id}`。
- `app/routers/dashboard.py` 完全替换：删除 markdown 文件 GET/PUT，仅保留 `GET /api/dashboard/stats` 聚合 4 个指标（消息数 / 媒体数 / 本月新增媒体 / 进行中 todo）。
- `app/config.py` 同步删除 `DASHBOARD_DEFAULT_CONTENT` 常量与 `get_dashboard_md_path()` 方法。

前端：
- 安装 `vuedraggable@^4`（基于 SortableJS）。
- `vue/src/types.ts` 新增 `Todo / TodoStatus / TodoBoard / DashboardStats` 类型。
- `vue/src/components/dashboard/` 四个组件：
  - `TodoCard.vue` — 单卡，双击编辑、hover 显示删除 ✕、done 状态删除线。
  - `TodoColumn.vue` — 单列，包含 vuedraggable list；pending 列底部多一个新增输入框。
  - `TodoBoard.vue` — 三列容器，处理 `vuedraggable` 的 `change` 事件 → 调 `PATCH /todos/{id}/move`，失败 toast + reload 回滚。
  - `RecentMedia.vue` — `/media?limit=12`，3-6 列响应式宫格。
  - `StatsCard.vue` — 调 `/api/dashboard/stats`，4 个数字卡 2x2 网格。
- `vue/src/views/Dashboard.vue` 完全重写为 widget 容器（之前 135 行 markdown 编辑器现在 24 行装配）。

**手测状态：**

后端 5 个端点已通过 curl 冒烟（端口 8003 起 worktree backend，因为用户日常 8002 还在跑旧代码）：
- POST 新建 → 出现在 pending[0]
- PATCH /move 跨列 → 进 doing[0]，pending 清空
- DELETE → 204
- /api/dashboard/stats → 4 个非负整数
- 旧 GET /api/dashboard → 404（端点已彻底移除）

service 层重排有独立 Python 烟测脚本，三条数据 ABC 跨列移动后 position 重排为 0..N-1，无空洞。

前端通过 `pnpm exec vite build` 成功打包（4.16s，Dashboard chunk 187KB）。`vue-tsc -b` 报 6 条 pre-existing 错误（TimelineBar / Actor / Media / router 历史 unused / implicit any），与本次改动无关，未修。

**未做浏览器手测**：前端 API base URL 写死指向 `:8002`（用户日常开发服务器，跑旧代码），验证整体路径需要在合并到 master 后启用本地完整环境，待人工验证以下流程：

1. 看板三列正确显示，最近媒体九宫格 + 4 个统计卡可见。
2. pending 列底部输入框，回车 → 卡片出现在 pending 列。
3. 拖到 doing 列 → 状态切换；刷新页面后保持。
4. 双击卡片改名，回车 → 标题更新；刷新后保持。
5. 拖到 done → 文本带删除线；hover ✕ 弹出确认对话框。
6. 拖拽到非法位置或断网 → toast 报错且看板回滚到上次状态。

## 主页活跃热力图（GitHub 风格）

**动机：** 主页只有数字统计太单薄，加一个 GitHub 提交日历那种一年视图，直观看出哪些时段写消息最频繁。

**改动：**

后端：
- `app/routers/dashboard.py` 新增 `GET /api/dashboard/heatmap`：返回 `{ start_date, end_date, days[365] }`，每天 `{ date, count }`，按 `created_at` 用 `strftime('%Y-%m-%d')` 分组聚合 Message 表。零计数日子也补齐，前端不用判空。

前端：
- `vue/src/components/dashboard/HeatmapCard.vue`：53 列 × 7 行格子，第一列向左 pad 到周日对齐；月份标签放在每周首日 ≤7 时显示；颜色 5 级（0 / 1-2 / 3-5 / 6-9 / 10+）走紫色色阶，配合现有暗紫渐变主题。横向 overflow-x-auto 防窄屏挤压。
- `vue/src/views/Dashboard.vue` 把 HeatmapCard 放在 TodoBoard 之前，作为页面顶部全宽 widget。

**未做：**

- 没做点击格子跳到当日消息，先看视觉够不够用。
- 未浏览器手测（前端 API base 仍指向 `:8002` 旧服务器，需手工启完整环境验证）。
