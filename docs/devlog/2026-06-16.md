# 2026-06-16

## feat(vue): 消息卡片支持 mosaic / grid 布局切换，默认 grid

### 背景

`MessageCard` 的多图(≥2 张)预览此前只有一种排布 —— Telegram 风格的 **mosaic**(`utils/mosaic.ts` 的 `calculateMosaicLayout` 按图片宽高比动态算 `rows` / `left_column`,每张图高度/宽度不一)。视觉丰富但不规整，浏览时不如等高方格整齐。

需求：多图也能用规整的等高网格(grid)排布，顶部工具栏加切换按钮在 mosaic / grid 间切，**默认 grid**，偏好持久化(跨会话记住)。单图保持原样(本就按宽高比显示、还有竖图限宽，grid 对它无意义)。

### 改动

1. **新增 composable `vue/src/composables/useMessageLayout.ts`**
   - 完全照搬 `useTheme.ts` 的「模块级单例 `ref` + localStorage」模式，所有消费方共享一份状态。
   - `localStorage` key = `message_layout`；`initLayout()` 读取，未知/空值一律回退 `grid`（默认）。
   - 暴露 `layout` / `setLayout` / `toggleLayout` / `initLayout`。

2. **`vue/src/views/Message.vue`**
   - 引入 `useMessageLayout`，`onMounted` 开头调 `initLayout()`。
   - 消息流工具栏（搜索 header）「合并」与「仅看收藏」之间加图标切换按钮，点击 `toggleLayout()`；grid 态显示「拼图」图标、mosaic 态显示「网格」图标（均提示点击后切到的目标布局），`title` 同步。
   - feed 区 `<MessageCard>` 透传 `:layout="messageLayout"`。右侧详情面板的媒体网格是页面内联模板、本就是 1/2/3 列方格，不动。

3. **`vue/src/components/MessageCard.vue`**
   - `Props` 加 `layout?: 'mosaic' | 'grid'`，`withDefaults` 兜底 `grid`。
   - 多图分支链改为：单图(`length === 1`，不变) → `v-else-if props.layout === 'grid'`（新增 grid 分支）→ 原 mosaic `rows` / `left_column`（不变）。
   - grid 分支：`grid gap-0.5` + 计算属性 `gridColsClass`（2 张 `grid-cols-2`，否则 `grid-cols-3`，与详情面板一致），每格 `aspect-square` + `object-cover`，复用现有的 video 角标 / 时长 / star 按钮 / 菜单（查找所有 message）/ `+N` overlay（仍挂在最后一格）。
   - 全程复用既有 `maxPreviewItems`(10)、`remainingCount`、`resolveThumb`、`isVideo`、`formatDuration`、`handleMediaClick`、`handleMediaToggleStar`、`toggleMenu`、`activeMenuIndex`、`findMessagesByMedia`、`mediaStarBouncing`，无新增逻辑。

### 验证

`cd vue && pnpm build`（`vue-tsc` 类型检查 + vite 构建）通过，仅有既存的 chunk 体积警告。默认 grid 显示规整方格，切换按钮在 mosaic / grid 间切并持久化到 localStorage，刷新保持；单图两种模式完全一致。

涉及文件：
- 新增：`vue/src/composables/useMessageLayout.ts`
- 修改：`vue/src/views/Message.vue`、`vue/src/components/MessageCard.vue`
