# 2026-05-29

## feat(web): Media 页面右侧时间轴重做（Apple Photos 风格）

### 背景
Media 页面右侧的 `DateScrubber` 时间轴与浏览器原生滚动条同时渲染、互相重叠，滚动条被时间轴 `z-30` 盖住完全点不到。同时旧时间轴只有细线 + 刻度，视觉表达较弱。

### 改动
- `vue/src/style.css`：新增 `.scrollbar-hidden` 工具类（`scrollbar-width: none` + `::-webkit-scrollbar { display: none }`），保留滚动能力。
- `vue/src/views/Media.vue`：主滚动容器追加 `scrollbar-hidden` 类；`<DateScrubber>` 接 `@wheel` 转发到 `scrollContainer.scrollBy({ top: dy })`，光标停在时间轴上也能正常滚列表。
- `vue/src/components/DateScrubber.vue` 重做：
  - 常驻宽度 `w-[28px] md:w-[40px]`，分"左标签区 26px + 右刻度轨道"两列布局。
  - 左标签区垂直分布年份文字；相邻 `top` 差 <8% 时折叠保留较新一年（刻度全部保留）。
  - 右轨道：中央细线 + 月份小刻度（1.5px）+ 年份长刻度（3px）。
  - 当前位置指示器升级：圆点（带光晕）+ 横向引出线 + "YYYY年M月" 毛玻璃胶囊标签；`timeline.length < 2` 或 <640px 屏宽隐藏胶囊。
  - 拖动 tooltip 改毛玻璃风格（`backdrop-blur-md`）。
  - 滚轮转发：`@wheel.passive` → `emit('wheel', deltaY)`。
  - 指示器/胶囊 `transition: top 200ms ease-out`，`prefers-reduced-motion` 下移除。

### 契约保持
`@jump` / `@jump-final` 事件未变；`useVirtualGrid` 完全未触碰；Smart 搜索模式（`smartActive=true`）的 `v-if` 守卫照旧。

### 验证
`vue-tsc --noEmit` 通过；浏览器实际表现需用户在 `pnpm dev` 后端到端测一遍（滚动条不可见 / 滚轮转发 / 拖动定位 / 亮暗主题 / 触屏模拟 / Smart 模式切换）。

### 相关
OpenSpec 变更：`openspec/changes/redesign-media-timeline/`（proposal / design / specs / tasks）。

## chore(web): 移除 Media 页面"回到最新"按钮

时间轴重做后定位已足够直观，移除右下角 `showBackToLatest` 浮动按钮（模板 + computed + handler 一并删除）。`scrollContainer` ref 与 `computed` 导入仍被其它逻辑使用，无需调整。

## style(web): Media 网格占满右侧内容区

去掉浏览模式与 Smart 搜索模式网格容器的 `max-w-4xl mx-auto` 限制，改为 `w-full`，让网格充满主内容区。浏览模式额外加 `pr-[32px] md:pr-[44px]`，避开右侧常驻时间轴宽度，单元格不会被遮挡。`useVirtualGrid` 用 `measureEl` 动态测量宽度计算列数，无需改动。

## fix(web): Media 网格列数动态化 + 时间轴留空修正

1. `useVirtualGrid.recomputeCellSize()` 改为基于 `measureEl` 宽度按目标单元格 220px 反推列数（`round(w/220)`），并保持最小 3 列（移动）/ 4 列（桌面）下限；移除 `onWindowResize` 里的硬编码 3/4 列；初始 `cols` 也改为占位 3，由首次测量决定。宽屏现在能自然变 5/6/7… 列。
2. 上一版把 `pr-[32px] md:pr-[44px]` 加到了 `measureEl` 上，`clientWidth` 仍包含 padding 导致单元格延伸到时间轴下。改为把右侧留白上移到 `scrollContainer`，`measureEl` 自身只剩对称的 `px-1 sm:px-2`，测出来的宽度就是真正可用区域。
