# 2026-05-13 开发日志

## 功能：新增 Issue 实体取代 Todo 看板,并接入 Message 过滤

### 背景

原 Todo 看板只承载短期任务,无法表达「一件长期推进的事」的语义——它需要有**原委**(为什么做)、**过程**(由一条条 message 累积串起)、**完成**(总结)。

Issue 模型与 Tag/Actor 平行,作为消息的第三类归属维度。

### 数据模型

- 新表 `issue`:`id / title / description(Markdown) / status / position / created_at / updated_at / completed_at`
  - `status` 四值:`doing` / `done` / `archived` / `abandoned`
- `Message` 新增 `issue_id INTEGER FK→issue.id ON DELETE SET NULL`(类比 `actor_id`)
- Alembic 迁移:`20260513_0900_5f1a2b3c4d5e_add_issue.py`
- SyncLog 追加 entity_type `ISSUE`,`_TRACKED_MODELS` 注册 `Issue`
