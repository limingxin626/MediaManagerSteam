# 2026-05-12 开发日志

## 功能：Vue 消息界面新建消息改为底部内联输入

### 背景
原本点击底部"写点什么..."按钮会弹出全屏 `MessageComposeDialog`，需要在弹窗内完成输入再点发送，操作链路偏重。希望改造为类似 Telegram/Slack 的底部即时输入条。

### 改动
- 新增 `vue/src/components/MessageComposeInline.vue`
  - 复用 `MilkdownEditor` + `useTagAutocompleteEditor`，支持 `#tag` 自动补全
  - 支持附件添加（Electron 走 `window.electronAPI.openFileDialog`，浏览器走 `<input type="file">`）
  - 已选标签 / 附件以 chip 行展示在输入框上方
  - 发送后自动清空内容、保留 `tagId` 预选项、重新聚焦编辑器
  - Props: `allTags` / `tagId` / `actorId`；Emit: `created`
- 修改 `vue/src/views/Message.vue`
  - 底部 placeholder 按钮替换为 `<MessageComposeInline>`，沿用 `mb-20 md:mb-0` 避让移动端 BottomNavBar
  - 删除 `openCreateDialog` 函数（不再使用）
  - 保留 `MessageComposeDialog`，仅服务于编辑场景（日期 / 媒体网格 / 拖拽排序未动）

### 验证
- `vue-tsc --noEmit` 通过
- 待用户在浏览器/Electron 中实测：基本发送、`#` 补全、附件添加、按 tag/actor 过滤上下文下发送

## 功能：编辑器从 Milkdown 切换到 Vditor（IR 所见即所得）

### 背景
Milkdown 的编辑体验偏 Notion，用户更喜欢 Obsidian 风格的源码/渲染混合（即时渲染 IR）。

### 改动
- 新增 `vue/src/components/VditorEditor.vue`，使用 Vditor 的 `mode: 'ir'`，工具栏 / 字数 / resize 全部隐藏，仅作为内联编辑器使用
- 对外暴露与 `MilkdownEditor` 完全相同的接口（`focus` / `getMarkdown` / `getCursorCoords` / `getTextBeforeCursor` / `deleteBeforeCursor` / `registerKeydown`），`useTagAutocompleteEditor` 无需改动
  - `getCursorCoords`：基于 `window.getSelection().getRangeAt(0)` 的 `getClientRects`
  - `getTextBeforeCursor`：从光标节点向前 DOM 遍历收集文本(到 editable 根为止)
  - `deleteBeforeCursor`：用 `selection.modify('extend','backward','character')` × N + `range.deleteContents()`，并补发 `input` 事件让 Vditor 同步内部 markdown
- `MessageComposeInline.vue` 中 `MilkdownEditor` 引用替换为 `VditorEditor`
- 保留 `MilkdownEditor.vue` 不删，便于回滚
- 新增依赖：`vditor ^3.11.2`

### 验证
- `vue-tsc --noEmit` 通过
- 待实测：IR 渲染效果、`#tag` 补全光标坐标、Enter/Tab 选词后删除前缀
