# 2026-05-06 开发日志

## Electron 下缩略图/头像走 file:// 本地绝对路径

**动机**：应用在 Electron 下纯本地运行，所有缩略图（`/data/thumbs/{id}.webp`）和头像（`/data/actor_cover/{id}.webp`）每次都从 FastAPI 静态托管走一次 HTTP 请求纯属浪费。Electron 主进程 `webSecurity: false` 早就允许 `<img>` 直接吃 `file:///E:/AskTao/thumbs/1.webp`，没理由不用。

**思路**：后端在 `MediaUrlMixin` / `ActorResponse` 的 schema 里**额外**带上文件系统绝对路径，前端在 Electron 环境下优先读这个字段并拼成 `file://` URL；浏览器/移动端读不到（或忽略）这个字段，继续走原来的 HTTP。后端静态挂载、resolveUrl 函数全部保留，零破坏性。

**改动**：

1. 后端 schema 新增字段（复用现有 `config.get_thumbnail_path` / `config.get_actor_avatar_path`）：
   - `backend/app/schemas/base.py`：`MediaUrlMixin` 加 `thumb_path: str = ""`，`_fill_urls` 中填充。`MessageMediaItem` / `MediaResponse` / `VideoPreviewItem` 通过继承自动获得。
   - `backend/app/schemas/actor.py`：`ActorResponse` 加 `avatar_abs_path: str = ""`（避开已有的 `avatar_path` ORM 字段名）。

2. 前端工具函数（`vue/src/utils/`）：
   - `constants.ts`：新增 `IS_ELECTRON = navigator.userAgent.includes('Electron')`。
   - `media.ts`：新增 `resolveThumb(item)` / `resolveAvatar(actor)`；Electron 下走 `file:///` + 反斜杠转正斜杠 + `#` 编码，浏览器下回落到 `resolveUrl(thumb_url/avatar_url)`。`thumb_url` 上的 `?t=...` 缓存戳会自动转移到 `file://` URL，让旋转/替换后的 cache-bust 仍然生效。
   - `types.ts`：手写的 `VideoPreviewItem` 加可选 `thumb_path`。

3. 替换所有 `<img>` 调用点（共 12 处）：`MediaCell` / `MediaPickerModal` / `MediaPreview`（缩略条带）/ `MessageCard`（4 处 mosaic）/ `MessageComposeDialog` / `VideoPreviewStrip` / `dashboard/RecentMedia` 改用 `resolveThumb`；`ActorCard` 改用 `resolveAvatar`。`MediaFeed.vue` / `Media.vue` / `RecentMedia.vue` 的 `openPreview()` 在构造 preview 切片时同步把 `thumb_path` 也透传，否则进预览看到缩略图栏会回落 HTTP。

4. **未改动**：
   - 后端 `/data/thumbs/...` 静态挂载保留，浏览器和 PWA 仍然要用。
   - `resolveUrl()` 函数本身不动，仍被 `MediaDetail` 的 video src、`Message.vue:245` 详情面板单点头像（仅有 actor_id 没有 actor 对象的场景）使用。
   - Electron 主进程、preload 没动；不需要新加 IPC，绝对路径完全由后端响应携带。
   - 视频文件 (`file_url`) 维持现状——`MessageCompose` 等已通过 `is_mounted_path` 处理了 Electron 直读本地视频。

**验证**：
- `pnpm vue-tsc --noEmit` 通过，前端类型完整。
- 后端 schema 改动是纯叠加字段（默认空字符串 + model_validator 填充），向后兼容，不需要前端先升级。
- 实际运行验证：浏览器 `http://localhost:5173` 缩略图仍走 HTTP；Electron `npm run dev` 下 DevTools Network 不再有 `/data/thumbs/`、`/data/actor_cover/` 请求，`<img src>` 形如 `file:///E:/AskTao/thumbs/1.webp`。这部分需要本地启 backend + electron 实测。
